# Hasta aquí (por ahora)

Si has llegado hasta esta página, **gracias de verdad**. Da igual si te has leído el libro de pe a pa o si solo has saltado a las preguntas que te caían en una entrevista: el objetivo era el mismo. Que salgas con más claridad, con mejores ejemplos y con un poco menos de nervios cuando alguien diga *«¿y useEffect?»*.

Espero que te haya gustado. Y si te ha servido para clavar una entrevista, para repasar un concepto que se te resistía o simplemente para tener una referencia a mano… **misión cumplida**.

## La web: reactjs.wiki

Todo este material también vive en la web:

→ [https://www.reactjs.wiki](https://www.reactjs.wiki)

Ahí puedes **buscar**, marcar lo que ya has leído y **poner a prueba** lo aprendido con los quizzes interactivos. Es cómodo para el día a día: abres el navegador, buscas la duda y listo.

La web y el libro se complementan. El libro va bien para estudiar con calma o sin conexión; la web, para consultar al vuelo y practicar.

## Es código abierto

Este proyecto es **open source**. El contenido, las preguntas, las respuestas y el código de la web están en GitHub:

→ [github.com/midudev/preguntas-entrevista-react](https://github.com/midudev/preguntas-entrevista-react)

Eso significa varias cosas buenas:

- Puedes **leer el código**, copiar ideas y aprender también de cómo está hecha la web.
- Si encuentras un error o echas en falta una pregunta, puedes **abrir un issue** o un pull request.
- El conocimiento no se queda encerrado: se puede mejorar entre todos.

Si te apetece contribuir, en el repo tienes la guía de contribución. No hace falta ser senior ni escribir un tratado: a veces un typo bien reportado o un ejemplo más claro ya suman mucho.

## Un último empujón

Las entrevistas de React no van de memorizar definiciones como un loro. Van de **entender** qué problema resuelve cada API, cuándo usarla y cuándo no. Si algo de este libro te ha acercado un poco más a eso, me doy por pagado.

Ahora toca lo importante: **practicar, construir cosas y equivocarse** (sí, también equivocarse; forma parte del pack).

Mucho ánimo con las entrevistas, con los proyectos y con lo que venga después.
